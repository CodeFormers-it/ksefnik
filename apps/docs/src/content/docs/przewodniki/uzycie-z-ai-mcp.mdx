Ksefnik działa jako serwer MCP (Model Context Protocol) — otwartego standardu komunikacji między asystentami AI a zewnętrznymi narzędziami. Dzięki temu możesz rozmawiać z AI o swoich fakturach KSeF i przelewach bankowych w języku naturalnym.

## Co to jest MCP?

Model Context Protocol to otwarty standard stworzony przez Anthropic. Definiuje sposób, w jaki asystent AI (np. Claude, Cursor) może wywoływać narzędzia zewnętrzne — pobierać dane, uruchamiać analizy, wykonywać akcje.

Ksefnik udostępnia 9 narzędzi MCP. Asystent AI sam decyduje, kiedy i które narzędzie wywołać, na podstawie Twojego pytania.

## Instalacja

Zero instalacji. Wystarczy `npx`:

```bash
npx @ksefnik/mcp
```

Klient AI (Claude Desktop, Cursor, Claude Code) uruchomi serwer automatycznie przy pierwszym użyciu.

## Konfiguracja

<Tabs>
  <TabItem label="Claude Desktop">
    Edytuj plik `~/Library/Application Support/Claude/claude_desktop_config.json` (macOS) lub `%APPDATA%/Claude/claude_desktop_config.json` (Windows):

    ```json
    {
      "mcpServers": {
        "ksefnik": {
          "command": "npx",
          "args": ["-y", "@ksefnik/mcp"],
          "env": {
            "KSEF_NIP": "1234567890",
            "KSEF_TOKEN": "eyJhbGciOi...",
            "KSEF_ENV": "demo"
          }
        }
      }
    }
    ```

    Po zapisaniu pliku zrestartuj Claude Desktop. Ksefnik pojawi się na liście dostępnych narzędzi.
  </TabItem>
  <TabItem label="Cursor">
    Utwórz plik `.cursor/mcp.json` w katalogu projektu:

    ```json
    {
      "mcpServers": {
        "ksefnik": {
          "command": "npx",
          "args": ["-y", "@ksefnik/mcp"],
          "env": {
            "KSEF_NIP": "1234567890",
            "KSEF_TOKEN": "eyJhbGciOi...",
            "KSEF_ENV": "demo"
          }
        }
      }
    }
    ```
  </TabItem>


    Lub dodaj do `.claude/settings.json`:

    ```json

          "command": "npx",
          "args": ["-y", "@ksefnik/mcp"],

    ```
  </TabItem>

### Zmienne środowiskowe

| Zmienna | Wymagana | Opis |
|---------|----------|------|
| `KSEF_NIP` | Tak | NIP podmiotu (10 cyfr) |
| `KSEF_TOKEN` | Nie | Token autoryzacyjny KSeF |
| `KSEF_ENV` | Nie | Środowisko: `prod`, `demo` lub `test` (domyślnie `demo`) |

Serwer akceptuje również aliasy: `KSEFNIK_NIP`, `KSEFNIK_TOKEN`, `KSEFNIK_ENV`.

<Aside type="tip">
Bez `KSEF_TOKEN` serwer automatycznie przełącza się na wbudowany symulator. To najszybszy sposób na przetestowanie integracji — nie potrzebujesz konta w KSeF ani tokena. Dane będą fikcyjne, ale pipeline działa identycznie.
</Aside>

## Dostępne narzędzia

Ksefnik udostępnia 9 narzędzi MCP:

| Narzędzie | Opis |
|-----------|------|
| `sync-invoices` | Synchronizacja faktur z KSeF |
| `query-invoices` | Wyszukiwanie faktur |
| `import-bank` | Import wyciągu bankowego |
| `reconcile` | Uzgadnianie faktur z przelewami |
| `get-unmatched` | Nieuzgodnione pozycje z raportu |
| `send-invoice` | Wysyłka faktury do KSeF |
| `validate-invoice` | Walidacja faktury |
| `confirm-match` | Potwierdzenie dopasowania |
| `get-upo` | Sprawdzenie statusu UPO (potwierdzenie odbioru) |

Szczegółowa dokumentacja każdego narzędzia: [Konfiguracja serwera MCP](/mcp/konfiguracja/).

## Przykładowa rozmowa

Poniżej typowy scenariusz — księgowa sprawdza uzgodnienie za marzec:

<Steps>
  1. Otwierasz Claude Desktop z podłączonym serwerem ksefnik.
  2. Piszesz: _"Pobierz faktury za marzec 2026 i zaimportuj wyciąg z pliku ~/Documents/wyciag-marzec.csv"_.
  3. Claude wywołuje `sync-invoices` z datami marca, potem `import-bank` z podaną ścieżką.
  4. Piszesz: _"Uruchom uzgadnianie i pokaż co nie pasuje"_.
  5. Claude wywołuje `reconcile`, potem `get-unmatched` i odpowiada:

     _"Uzgodniłem 12 z 15 faktur (match rate 80%, średnia pewność 93%). 3 faktury bez przelewu:_
     - _FV/2026/03/012 — 4 500 PLN (ACME sp. z o.o.)_
     - _FV/2026/03/018 — 1 230 PLN (TechSoft)_
     - _FV/2026/03/021 — 8 900 PLN (DataCorp)"_
  6. Piszesz: _"Sprawdź UPO dla faktury KSeF-1234567890-20260310-ABC123"_.
  7. Claude wywołuje `get-upo` i potwierdza status odbioru.
</Steps>

## Przykładowe prompty

### Podstawowe operacje

- _"Pobierz faktury za marzec 2026 i pokaż podsumowanie"_
- _"Zaimportuj wyciąg bankowy z pliku wyciag-marzec.csv"_
- _"Uruchom uzgadnianie za marzec z wyciągiem z mBanku"_

### Analiza danych

- _"Które faktury z marca nie mają przelewu?"_
- _"Jaka jest łączna kwota niesparowanych faktur?"_
- _"Pokaż faktury od kontrahenta z NIP 7740001454"_
- _"Jaki jest match rate za marzec?"_

### Zaawansowane pytania

- _"Porównaj match rate z lutego i marca — czy się poprawił?"_
- _"Znajdź faktury powyżej 10 000 PLN, które nie zostały opłacone"_
- _"Zwaliduj fakturę FV/2026/03/042 przed wysyłką do KSeF"_
- _"Sprawdź UPO dla ostatnio wysłanej faktury"_

<Aside type="note">
AI nie wysyła danych do zewnętrznych serwerów. Komunikacja MCP odbywa się lokalnie — między klientem AI a procesem ksefnik na Twoim komputerze.
</Aside>
